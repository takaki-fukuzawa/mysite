git add ステージに登録するファイル

git add -p ファイル

git commit -m メッセージ

git rm 削除するファイル

git checkout ブランチ名

git clone クローンするリポジトリURL

git push リモートリポジトリ　プッシュするブランチ

git pull リモートリポジトリ　プルするブランチ

フォークしたリポジトリから管理者のアカウント（中央リポジトリ）に
プルリクエスト（私の変更を反映してください）というアクションをしている。

プルリクエスト
→フォークしたリポジトリからのコミットについて、元のリポジトリへ取り込むことを依頼すること。

git remote add

コミットはひとつの作業ごとに行う
基本的にはひとつの作業を終えるごとにコミットすることが望ましい。
コミット単位でしかファイルの復元ができないため、多くの作業を同時にコミットするとそれらを全て戻さないといけないケースも発生する。
